Sprint Conclusions

Summary

This first Sprint (Sprint0) grouped the requirements of the project, by outlining the following macro components, ordered by priority:
  1. Core-business (requirements 1 & 2): involves the chain of the main nodes of the system (smart device, waste service, transport trolley, basic robot).
    High priority since the system cannot provide the service without it.
  2. RaspberryPi (requirement 4): includes alarm (sonar) and warning (led) elements.
    Medium priority since it involves the security of the service.
  3. Monitoring (requirement 3): involves the GUI that display information on the current status of the service.
    Low priority since it can be seen as a secondary, fancier and remote output (in addition to the textual output), and depends on the other 2 components.

Work Plan

We decided to partition the system into 3 main sprints, which follows the macro components mentioned above.

Sprint1 - Core-Business

The Sprint1 will focus on the Core-business, which is also the first requirement, and is going to include:
  • Dummy simulator of TruckDriver + SmartDevice (with a test GUI).
  • WasteService: problem analysis including a model for "advanced" interactions, waste service area in-depth analysis, project and implementation of a prototype.
  • TransportTrolley: problem analysis, project and prototyping of the element that will control the robot movements.
  • BasicRobot: the software is ready and serviceable, so it just has to be integrated in the project.
