General

Sections

The problem analysis will be subdivided in a hierarchy of sections, denoted by colored boxes:

  • Part of the current Sprint

    Only one per file, it represents the document title.

  • Macro topic

    Groups one or many sub-topics.

  • Sub-topic

    Concerns a small topic.

  • Miscellaneous, extras or related content

    Contains extra details or additional information.
  • Single important parts
    Used to remark important parts, independently from the section/topic location.

Problem Analysis

Along the analysis, we'll point out some problems which needs to be analysed, discussed and for wchich one or multiple solution/s has/have to be found.

In general, for each topic or problem, we will have the following blocks:
  • The problem This is usually followed by a series of questions.
  • Considerations, possible solutions or alternatives Contains the densest part of the analysis, where our team discusses and does brainstorming to try to gather and address each possible aspect of a problem.
  • One or more solution described more or less in detail Highlights the solution/s that we are going to adopt or try to implement in the following phases of the sprint.
