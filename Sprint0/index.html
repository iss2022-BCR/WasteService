Sprint 0

Introduction

Sprint0 focuses on the analysis of requirements provided by the customer. Our task is to fully comprehend them, by clarifying doubts and uncertainties, so that everyone (team members and the customer/product owner) agrees on their meaning.

In particular, Sprint0 aims:
  • to formalize requirements, in order to make them machine-understandable;
  • to draft one model of a general overview of the system, divided by computational nodes;
  • to draft a first-primitive interaction scheme;
  • to provide some executable test plans, which will serve to show the customer the main behaviour of the system, in order to discuss the correctness and eventual issues/differences.
Since Sprint0 doesn't discuss any problem yet, the problem analysis section will be empty.

At the end of the Sprint0, we will draft some conclusion and provide a work plan which will schedule the following Sprints. Especially, we're going to outline the core-business of the project.

Motto

There is no code without a project, there is no project without problem analysis, there is no problem without requirements.
