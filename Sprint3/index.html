Sprint 3

Introduction

Our team must project and develop a software system to manage a waste-disposal service, as described in Final Theme 2022.

Scrum Goal

We must develop a graphic application WasteServiceStatusGUI capable of displaying information about the WasteService (requirement 3). In particular, the GUI must display:
  • the current state of the TransportTrolley and its position in the room;
  • the current weight of the material stored in the two waste-containers;
  • the current state of the Led;
We also have to provide the customer an executable demo of the component being integrated with the rest of the system.

Work Plan

  • Analyze what's needed to implement the GUI in order to meet the customer requirements
  • Find and address any problem related to the requirement (find difficulties, make considerations, suggest solution/s)
  • Implement the GUI
  • Implement some API the GUI can use to retrieve the WasteService information, or some way to integrate the GUI with the rest of the system
  • Package the project and create an easy way to deploy and launch it
  • Create a demo of the whole system
NB: eventually make use of the software provided by our software house.

Motto

There is no code without a project, there is no project without problem analysis, there is no problem without requirements.
