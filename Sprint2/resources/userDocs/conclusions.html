Sprint Conclusions

Summary

During Sprint 2 our team analyzed and implemented the software needed to meet the 4th requirement. The software was mostly related to the Raspberry Pi part of the system, and we believe that our solution has many points of strength, in particular:
  • the possibility to use a real Raspberry Pi device with components, or to use a simulation;
  • presents extended features: display and buzzer;
  • provides a wide choice of scripts to automatize the building and deployment processes, as well as the execution;
  • is easily extendable.
We tested the WasteService system with different architecture configurations:
  • Simulated environment in LAN: SmartDevice emulated on pc machine, WasteService Core on pc machine, WasteService Rpi on pc machine.
  • Real environment in LAN: SmartDevice on smartphone, WasteService Core on pc machine, WasteService RPi on Raspberry Pi device.
  • Real environment over the Internet: SmartDevice on smartphone (connected to a cellular connection), WasteService Core on pc machine (connected to router1), WasteService Rpi on Raspberry Pi device (connected to router2).
The output of the Sprint2 is the implementation of the following softwares:
  1. WasteService Core
  2. RadarSystem22.domainBCR
  3. WasteService RPi

Setup

PC Node
  1. Install Java (>8 or >11 if one wants to use the MapEditor);
  2. Install docker.
  3. Install gradle (>= 7.4).
  4. (Optional, if one doesn't have an Android device)
    • Install an Android Emulator (guide);
    • Install Flutter (guide)
  5. From the latest release on our GitHub repository, download:
    • VirtualRobot docker image (virtualRobotOnly4.0.yaml).
    • WasteService ZIP archive (waste_service.zip).
    • SmartDevice APK (smart_device.apk).
    NB: file names can vary a bit (sprint number, version number). Use the latest version available.
